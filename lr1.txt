5 права доступа к обьектам
смена прав доступа кобьектам выполняется администратором chmod
sudo chmod 775 <imya objecta>
sudo chmod g-x <imya>

смена владельцев и группы владельа выполняется админом командой chown

Режим группового доступа к обьектам можно использовать при смене группы,заданной поумолчанию

специальные права доступа к обьектам SUID, GUID, sticky bit


Специальный бит SUID предназначен для запуска файла от имени нго владельца.Эта процедура позволяет пользователям получить права на доступ к обьектам ,
которыми может распоряжаться владелец файла.
Пример - /bin/passwd для функционирования которой нужны права записи в файл
/etc/shadow
Такие права есть только у пользователя root поэтому у программы /bin/passwd
владелец руут и установлен бит SUID
В результате любой пользователь может сменить пароль без администратора

Установка бита SUID осуществляется при помощи программ chmod:
sudo chmod u+s <imya faila>
sudo chmod 4775 <imya>

сброс бита SUID аналогично 
sudo chmod u-s <imya faila>
sudo chmod 0775 <imya>

Бит SUID нет смысла применять к папкам.

Специальны бит GUID предназначен для запуска файла от имени группы владельцев 
или для изменения правил наследования группы владельцев
 если он установлен для папки
администратора

Установка бита GUID осуществляется при помощи программ chmod:
sudo chmod G+s <imya faila>
sudo chmod 2775 <imya>

Права доступа к файлу с установленым GUID

-rwxrwsr-x 1 student root       0 окт 30 12:58  f1

Права доступа к каталогу  с установленым GUID
drwxrwsr-x 3 student root    4096 окт 30 13:19 F

При установке бита GUID на папку все обьекты созданные внутри папки наследуют группу владельцев от родителя
при этом дочернии папки наследуют бит GUID
